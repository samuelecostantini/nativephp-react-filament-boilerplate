# Analysis: 405 Method Not Allowed on Route `/` (NativePHP Android)

## Problem Description
When opening the Filament admin panel in the native Android application, the app responds with an `error 405: the method is not supported for route /. Supported methods: get, head.`

## Root Cause Analysis
Based on the codebase investigation, the following factors contribute to this error:

1.  **`APP_URL` Mismatch:** The current `APP_URL` in the configuration is set to `https://totem-ariston-android.test:8000`. In a NativePHP Android environment, the app is served locally (usually on `http://127.0.0.1` or `http://localhost`). Absolute URLs generated by Laravel (for redirects, assets, or bridge calls) use the `APP_URL` from the `.env`. If the WebView doesn't correctly resolve this domain or if there's a protocol mismatch (HTTPS vs HTTP), it can lead to routing failures.
2.  **CSRF Token Failures:** Because of the `APP_URL` and domain mismatch, the CSRF token cookie might not be correctly associated with the domain the WebView is actually using. This causes POST requests (like Livewire updates or Native bridge calls) to fail the CSRF check.
3.  **Redirected POST Requests:** When a CSRF check fails or an authentication error occurs during a POST request (e.g., an automated Livewire update or a Native bridge call), Laravel might be configured to redirect to the home page (`/`). If this redirect is handled as a `307 Temporary Redirect`, the browser will attempt to POST to the new location (`/`). Since the root route only supports `GET` and `HEAD`, this results in the 405 Method Not Allowed error.
4.  **Livewire/Filament Initialization:** Filament 3 and Livewire 3 might trigger automated POST requests on page load for synchronization or polling. If the base URL is miscalculated due to the `APP_URL` issue, these requests might hit the wrong endpoint or trigger redirects.

## Proposed Fix Strategy
1.  **Correct the `APP_URL`:** The `APP_URL` in the `.env` should be set to `http://localhost` (or a dynamic value) for the native build to ensure absolute URLs match the local server.
2.  **Verify CSRF Exclusions:** Ensure that all internal NativePHP bridge routes (e.g., `_native/*`) are excluded from CSRF verification.
3.  **Filament Panel Configuration:** Ensure the Filament panel is marked as `default()` and that its pathing is compatible with the local server environment.
4.  **Inertia vs Native Navigation:** Ensure that the navigation from the Inertia home page to the Filament panel is a clean full-page GET request.
